# Security Policy

Xenova Labs CBC takes the security and privacy of Harmony users seriously.

## Supported versions

Security updates are applied to the latest version on the `main` branch. Older
commits, forks, and modified distributions are not supported by Xenova Labs CBC.

## Reporting a vulnerability

Please report suspected vulnerabilities privately by emailing
[contact@xenovalabs.com](mailto:contact@xenovalabs.com) with the subject
`Harmony security report`.

Include, where possible:

- a clear description of the issue and its potential impact;
- the affected browser, operating system, version, or commit;
- reproducible steps or a minimal proof of concept; and
- any suggested remediation or relevant supporting material.

Do not include credentials, personal data, or camera recordings unless Xenova
Labs CBC explicitly requests them through an agreed secure channel. Do not open
a public GitHub issue for an unpatched vulnerability.

Xenova Labs CBC will acknowledge the report, investigate it, and coordinate an
appropriate disclosure timeline based on severity and complexity. Please allow
reasonable time for remediation before publishing details.

## Safe-harbor intent

Good-faith research that avoids privacy violations, service disruption, data
destruction, and unauthorized access beyond what is necessary to demonstrate a
finding will be treated as authorized for the purpose of reporting the issue.
This statement does not waive the rights of third parties or authorize testing
systems outside Harmony's scope.
